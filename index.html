Esta proposta tem como objetivo estruturar o lançamento inicial do negócio digital, permitindo a validação de mercado com rapidez, baixo risco e base preparada para evolução futura. O foco não é apenas construir um produto, mas viabilizar o início da operação com potencial de crescimento estruturado.

Proposta 01

Produto Completo

Investimento total
R$ 300.000
Design de produto + Desenvolvimento
12 meses

Nesta modalidade, o produto é construído do zero até a v1 completa ao longo de aproximadamente 8 a 12 meses. O trabalho acontece em ciclos chamados sprints — períodos de 2 semanas onde a equipe entrega partes do produto de forma incremental. O processo funciona assim: enquanto o time de design valida fluxos, testa hipóteses e refina a experiência com usuários reais, o time de desenvolvimento vai codando o que foi aprovado logo atrás. Tudo acompanhado e organizado via ferramentas como ClickUp, com reuniões periódicas de alinhamento, revisão de entregas e ajuste de prioridades. É a forma mais completa de construir um produto — mas também a que exige maior investimento de capital antes de qualquer validação de mercado. Você estará construindo tudo antes de saber o que os usuários realmente precisam.

Este é o escopo completo de um produto no nível do Quinto Andar. São mais de 80 funcionalidades envolvidas — construir tudo isso de uma vez, sem validação prévia, representa alto risco de capital e de direção.

Proposta 02

MVP Isolado

Investimento fixo
R$ 60.000
MVP via Vibe Code · 2 meses
Sem acompanhamento posterior

Um MVP — Minimum Viable Product — é a versão mais enxuta do produto que ainda entrega valor real para o usuário. A ideia é lançar rápido, com o essencial funcionando, e usar o comportamento dos usuários reais para guiar as próximas decisões. Nesta opção, você sai com o produto no ar em 2 meses. O que não está incluído: o acompanhamento de métricas, a análise de como os usuários estão usando o produto, e a definição do roadmap para construir a v1 robusta. Esses passos — que são críticos para não errar na direção do produto — ficam por conta do cliente após a entrega.

Escopo — 4 blocos de entrega

Estratégia de Produto e Negócio
  • Definição do fluxo principal
  • Estruturação da operação inicial (manual + digital)
  • Recorte do MVP (priorização do essencial)
Branding Essencial
  • Nome (validação e ajustes)
  • Identidade visual (logo e base visual)
  • Diretrizes de aplicação
Plataforma Web MVP (via Vibe Code)
  • Busca de imóveis (cidade, bairro, preço)
  • Listagem de imóveis
  • Página de imóvel com fotos, preço e descrição
  • Botão de agendamento de visita
  • Formulário de agendamento (nome, telefone, data desejada)
  • Envio automático para planilha + notificação (email ou WhatsApp)
  • Gestão via planilha, distribuição manual de visitas
  • Cadastro de imóveis via formulário interno ou planilha
Base Estrutural (Design System Leve)
  • Componentes reutilizáveis
  • Padrões de layout
  • Estrutura pensada para evolução futura

Esta opção entrega o MVP sem acompanhamento posterior. O próximo passo — estruturar o roadmap, analisar o comportamento dos usuários e preparar a v1 — ficará por conta do cliente.
